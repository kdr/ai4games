Luigi Bros Troubleshooting Guide

This guide will help you diagnose and fix common issues with the Luigi Bros game. If you're experiencing problems, work through these steps to get your game running.

Back to Game Asset Test

Issue: Blank Screen / Game Not Loading

1. Check Console for Errors

Open your browser's developer tools by pressing F12 or Ctrl+Shift+I (Windows/Linux) or Cmd+Option+I (Mac). Look for any red error messages in the console.

2. Common Error Messages and Solutions

"Phaser library not loaded"

Solution: Ensure you have an internet connection to load the Phaser library from CDN. Alternatively, download Phaser locally and update the script tag in index.html.

"Scene class not found"

Solution: Make sure all scene JS files are correctly loaded before config.js. Check the script tags in index.html to verify they're in the correct order.

"Failed to load asset"

Solution: Make sure all image and sound files are in the correct directories:
  • Images should be in: assets/images/
  • Sounds should be in: assets/sounds/
Run the Asset Test to check which files are missing or cannot be loaded.

Issue: Game Loads But Doesn't Work Correctly

1. No Controls / Cannot Move Character

Solution: Make sure keyboard controls are working by testing the arrow keys. On mobile devices, ensure touch controls are visible and responsive.

2. Game Is Too Slow / Laggy

Solution: Try reducing the game size or switching to a more powerful device. In config.js, you can set pixelArt: false to improve performance.

Browser Compatibility

The game works best on modern browsers. If you're using an older browser, try updating to the latest version or switching to Chrome, Firefox, or Edge.

Local Server

Some browsers restrict loading assets when opening HTML files directly. Try running the game on a local server: 

# Using Python 3
python -m http.server

# Using Node.js
npx http-server

Then access the game at http://localhost:8000 or similar.

Contact Support

If you've tried all the above steps and still can't get the game working, please contact support with details about your issue, including:

  • Browser name and version
  • Device type (desktop/mobile)
  • Error messages from the console
  • A screenshot of the issue
